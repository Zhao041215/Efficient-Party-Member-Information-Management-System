## 第一章 系统简介

### 1.1 系统概述与超级管理员职责

生化学院党员信息管理系统的超级管理员（superadmin）是系统权限最高的角色，是整个系统的最终管理者。超级管理员负责：

- **全角色账户管理：** 创建、管理学生、教师、管理员的所有账户（普通管理员只能管理学生账户）
- **信息审核：** 审核学生提交的信息修改申请
- **数据管理：** 查看、导出、导入学生和毕业生数据
- **系统设置：** 维护全部系统选项（学院、年级、班级、政治面貌、发展时间）
- **反馈管理：** 处理全体用户提交的 Bug 反馈和功能建议（普通管理员无此权限）
- **日志监控：** 查看所有用户的操作日志，监控系统使用安全
- **数据总览：** 在数据展板查看包含角色人数分布在内的全面统计数据

超级管理员是系统中**唯一能够创建和管理教师账户及管理员账户**的角色，同时也是处理用户反馈的唯一责任方。

### 1.2 与普通管理员的权限区别

超级管理员拥有普通管理员的全部权限，并在以下方面享有额外权限：

| 功能 | 普通管理员 | 超级管理员 |
|------|-----------|-----------|
| 创建学生账户 | ✅ | ✅ |
| 创建教师账户 | ❌ | ✅ |
| 创建管理员账户 | ❌ | ✅ |
| 管理学生账户（重置密码/禁用/删除） | ✅ | ✅ |
| 管理教师账户 | ❌ | ✅ |
| 管理管理员账户 | ❌ | ✅ |
| 批量导入账户 | ✅（仅学生） | ✅（全角色） |
| 信息审核 | ✅ | ✅ |
| 学生信息管理 | ✅ | ✅ |
| 毕业生管理 | ✅ | ✅ |
| 系统设置 | ✅ | ✅ |
| 操作日志查看 | ✅ | ✅ |
| 反馈管理 | ❌ | ✅ |
| 数据展板角色人数分布 | ❌ | ✅ |
| 左侧菜单中的"反馈管理"入口 | ❌ 不显示 | ✅ 显示 |

> ⚠️ **重要提示：** 超级管理员账户权限极高，请务必：
> - 设置高强度密码，定期更换
> - 不在公共设备上登录，使用后及时退出
> - 不将账号信息告知他人
> - 定期查看操作日志，确认账号未被异常使用

### 1.3 浏览器与设备要求

| 浏览器 | 推荐版本 |
|--------|----------|
| Google Chrome | 90 及以上 |
| Microsoft Edge | 90 及以上 |
| Firefox | 88 及以上 |
| Safari | 14 及以上 |

建议在安全的个人电脑上使用，不建议在公共电脑上登录超级管理员账户。

## 第二章 账号与登录

### 2.1 首次登录与强制修改密码

超级管理员账户通常由系统部署时在数据库中直接创建（初始用户名：`admin`，初始密码：`admin123`），或由上级管理员告知。首次登录必须修改密码：

**第一步：** 打开浏览器，访问系统登录页面。

**第二步：** 在【角色】下拉菜单中，点击并选择【系统管理员】（即超级管理员对应的登录角色）。

**第三步：** 在【用户名】输入框中输入超级管理员用户名（如"admin"或系统部署时设定的用户名）。

**第四步：** 在【密码】输入框中输入初始密码（如"admin123"或告知的初始密码）。

**第五步：** 点击【登录】按钮。

**第六步：** 系统检测到需要修改密码，自动跳转至修改密码页面。

**第七步：** 在【当前密码】输入框中输入初始密码。

**第八步：** 在【新密码】输入框中输入强密码（**强烈建议**：至少12位，包含大写字母、小写字母、数字、特殊符号）。

**第九步：** 在【确认新密码】输入框中再次输入相同密码。

**第十步：** 点击【修改密码】按钮，修改成功后自动跳转至数据展板。

> ⚠️ **安全警告：** 超级管理员初始密码（admin123）是公开已知的，必须在首次登录后**立即修改**，否则任何人都可能以超级管理员身份登录系统，造成严重的数据安全风险。

### 2.2 日常登录步骤

**第一步：** 访问系统登录页面，在【角色】下拉框中选择【系统管理员】。

**第二步：** 输入用户名和密码，点击【登录】按钮，登录成功后跳转至数据展板。

> ⚠️ **注意：** 不建议在公共电脑上勾选"记住我"，以防账号被他人使用。

### 2.3 忘记密码与重置密码

**前提：** 账号已绑定邮箱（首次登录后建议先绑定邮箱）。

**第一步：** 在登录页面点击【忘记密码？】链接。

**第二步：** 输入账号绑定的邮箱地址，点击【发送验证码】。

**第三步：** 查收邮件中的 6 位验证码（有效期 10 分钟）。

**第四步：** 输入验证码，点击【验证】。

**第五步：** 设置新密码并确认，点击【重置密码】，重置成功后重新登录。

> ⚠️ **重要：** 若超级管理员账号未绑定邮箱且忘记密码，无法通过系统自助找回。需要直接操作服务器数据库重置密码，请联系系统部署人员或服务器管理员（详见 [12.4 超级管理员账号密码忘记怎么办](#124-超级管理员账号密码忘记怎么办)）。

### 2.4 退出登录

点击左侧菜单底部或右上角的【退出登录】按钮，系统清除登录状态，返回登录页面。每次使用完毕后务必退出。

## 第三章 数据展板

### 3.1 进入数据展板

登录后系统默认进入数据展板页面。也可随时点击左侧菜单顶部的【数据展板】进入。

超级管理员的数据展板在普通管理员基础上，展示更多维度的统计信息。

### 3.2 超级管理员专属统计卡片

除普通管理员可见的统计卡片外，超级管理员数据展板还额外显示**各角色账户人数分布**统计，具体卡片如下：

**全部统计卡片（超级管理员可见）：**

| 卡片名称 | 含义说明 |
|----------|----------|
| 在校学生总数 | 当前状态正常的在校学生总人数 |
| 正式党员 | 政治面貌为"中共党员"/"正式党员"的学生人数 |
| 预备党员 | 政治面貌为"中共预备党员"/"预备党员"的学生人数 |
| 入党积极分子 | 政治面貌为"入党积极分子"的学生人数 |
| 待审核申请 | 当前等待审核的学生信息修改申请总数 |
| 毕业生档案 | 已归档的毕业生总人数 |
| 系统账户总数 | 系统中所有角色的账户总数量 |
| **学生账户数** | **（超级管理员专属）** 系统中学生角色账户的数量 |
| **教师账户数** | **（超级管理员专属）** 系统中教师角色账户的数量 |
| **管理员账户数** | **（超级管理员专属）** 系统中管理员角色账户的数量 |
| **系统管理员数** | **（超级管理员专属）** 系统中超级管理员角色账户的数量 |

通过这些专属卡片，超级管理员可以快速掌握各角色账户的规模，判断是否需要新增或调整人员配置。

### 3.3 筛选与图表功能

**筛选操作：**

**第一步：** 在展板顶部，找到年级、班级、政治面貌、性别等筛选下拉框。

**第二步：** 点击需要筛选的下拉框，选择条件。

**第三步：** 页面自动刷新，所有统计卡片（包括超级管理员专属卡片）和图表均按所选条件更新。

**第四步：** 将所有筛选条件重置为"全部"，恢复全量数据。

**6 个统计图表（与普通管理员相同）：**

| 图表序号 | 图表类型 | 内容说明 |
|----------|----------|----------|
| 图表一 | 甜甜圈图 | 各政治面貌的人数占比 |
| 图表二 | 柱状图 | 各年级学生人数对比 |
| 图表三 | 柱状图 | 各班级学生人数对比 |
| 图表四 | 饼图 | 男女生比例 |
| 图表五 | 水平条形图 | 民族分布 Top 10 |
| 图表六 | 折线图 | 历年党员发展趋势 |

鼠标悬停在图表的任意数据点或扇形上，可查看该项的精确数字和占比。

### 3.4 实时数据更新

数据展板的统计卡片每隔约 **5 秒**自动刷新。以下内容会实时更新：
- 所有统计数字卡片（含角色人数分布）
- 左侧菜单【信息审核】旁的红色数字徽章

**无需手动刷新页面**，数据始终保持最新状态。

## 第四章 账户管理（全角色权限）

### 4.1 超级管理员账户管理权限说明

超级管理员在账户管理页面可以看到**所有角色**的账户（学生、教师、管理员、超级管理员），并对其中任何账户执行管理操作。

**重要权限对比：**

| 操作 | 普通管理员能操作的角色 | 超级管理员能操作的角色 |
|------|----------------------|----------------------|
| 新增账户 | 仅学生 | 学生、教师、管理员、超级管理员 |
| 重置密码 | 仅学生 | 全部角色 |
| 启用/禁用 | 仅学生 | 全部角色 |
| 学籍变动 | 仅学生 | 仅学生（教师/管理员无学籍变动概念） |
| 删除账户 | 仅学生 | 全部角色 |

> ⚠️ **警告：** 超级管理员账户权限极高，在管理其他管理员账户时，请格外谨慎，误操作可能导致系统管理功能不可用。

### 4.2 新增教师账户

**第一步：** 点击左侧菜单中的【账户管理】，进入账户管理页面。

**第二步：** 点击右上角的【添加账户】按钮，弹出"新增账户"模态框。

**第三步：** 在【用户名】输入框中输入教师的工号（或其他唯一标识符，如"T2024001"）。

**第四步：** 在【姓名】输入框中输入教师的真实姓名。

**第五步：** 在【角色】下拉框中选择【教师】。

**第六步：** 选择教师角色后，性别字段变为可填项，在【性别】下拉框中选择"男"或"女"。

> ⚠️ **说明：** 教师账户无需填写学院、年级、班级（这些是学生专属字段），选择教师角色后这些字段不会出现。

**第七步：**（可选）在【初始密码】输入框中设置该账户的初始密码。若不填写，初始密码默认为用户名（工号），且账户被标记为"首次登录必须修改密码"。

**第八步：** 点击【提交】按钮，系统创建教师账户，提示"账户创建成功"。

**第九步：** 将用户名（工号）和初始密码告知该教师，提醒其登录后立即修改密码。

> ⚠️ **注意：** 
> - 教师账户用户名一旦创建不可修改，请仔细核对工号
> - 若需要为教师账户绑定邮箱（用于密码找回），需教师自行登录后在修改密码页面操作，或通过后台数据库直接设置

### 4.3 新增管理员账户

**第一步：** 在账户管理页面，点击【添加账户】按钮。

**第二步：** 在模态框中，在【用户名】输入框中输入管理员的账号名（如"admin2024"或该管理员的工号）。

**第三步：** 在【姓名】输入框中输入管理员的真实姓名。

**第四步：** 在【角色】下拉框中选择【管理员】。

**第五步：** 在【性别】下拉框中选择"男"或"女"。

**第六步：**（强烈建议）在【初始密码】输入框中设置一个相对复杂的初始密码，并单独告知该管理员，而非使用默认的"用户名=密码"。

**第七步：** 点击【提交】按钮，系统创建管理员账户，提示"账户创建成功"。

**第八步：** 将账号信息告知该管理员，并要求其首次登录后立即修改密码。

> ⚠️ **重要提示：**
> - 管理员账户具有较高权限（可管理全部学生账户、审核信息、系统设置等），应谨慎分配
> - 建议同时将新管理员的账户信息记录在线下文档中，以备不时之需
> - 若管理员人员变动（如离职），应及时禁用或删除其账户

### 4.4 新增学生账户

与普通管理员操作完全相同，步骤如下：

**第一步：** 点击【添加账户】按钮，在模态框中填写：
- **用户名：** 学生学号
- **姓名：** 真实姓名
- **角色：** 选择【学生】
- **性别：** 男/女
- **学院：** 从下拉框选择（必须是系统设置中已有的学院）
- **年级：** 从下拉框选择（必须是系统设置中已有的年级）
- **班级：** 从下拉框选择（必须是系统设置中已有的班级）

**第二步：**（可选）设置初始密码，若不填默认为学号。

**第三步：** 点击【提交】，账户创建成功。

### 4.5 管理教师账户

超级管理员可以对教师账户执行以下操作，这些操作在普通管理员处均不可用：

**重置教师密码：**

**第一步：** 在账户管理页面，通过角色筛选找到教师账户（在【角色】筛选框中选择"教师"）。

**第二步：** 找到需要重置密码的教师账户所在行，点击【重置密码】按钮。

**第三步：** 系统弹出确认框，要求输入**您（超级管理员）自己的当前登录密码**进行二次验证。

**第四步：** 输入您自己的密码后，点击【确认重置】。

**第五步：** 系统将该教师账户密码重置为其用户名（工号），并标记"下次登录必须修改密码"。

**第六步：** 通知该教师使用工号作为密码重新登录，并提醒其立即修改密码。

---

**禁用教师账户：**

**第一步：** 找到目标教师账户所在行，状态显示为绿色"启用"。

**第二步：** 点击【禁用】按钮，系统弹出确认框。

**第三步：** 点击【确认】，该教师账户立即无法登录。

> ⚠️ **使用场景：** 当教师休假或暂时不使用系统时，可禁用账户防止非授权访问；需要恢复时点击【启用】即可。

---

**启用教师账户：**

找到状态为红色"禁用"的教师账户，点击【启用】按钮，确认后恢复正常登录权限。

---

**删除教师账户：**

**第一步：** 找到需要删除的教师账户所在行，点击【删除】按钮。

**第二步：** 系统弹出二次确认对话框，提示"确定删除该账户吗？此操作不可撤销。"

**第三步：** 点击【确认删除】，系统删除该教师账户。

> ⚠️ **注意：** 删除教师账户仅删除账户本身，不影响系统中的其他数据（教师不关联学生信息数据）。若仅是人员离职，建议使用【禁用】而非删除，以保留账户历史记录。

### 4.6 管理管理员账户

超级管理员可以对普通管理员账户执行重置密码、启用/禁用、删除操作，操作步骤与管理教师账户完全相同（参见 4.5 节），区别在于筛选角色时选择"管理员"。

> ⚠️ **特别注意：**
> - 不建议轻易删除管理员账户，这会导致其历史操作日志中的账户关联信息缺失
> - 超级管理员可以禁用或重置其他管理员的密码，但请确保系统中**至少有一个超级管理员账户处于启用状态**，否则将无人能够执行超级管理员专属操作
> - 超级管理员无法对另一个超级管理员账户执行某些操作（系统有保护机制防止互相锁定）

### 4.7 管理学生账户

与普通管理员操作完全相同，超级管理员可对学生账户执行全部操作：
- 重置密码（需输入自己密码二次确认）
- 启用/禁用账户
- 学籍变动（毕业归档 / 转专业班级）
- 删除账户

详细操作步骤请参阅《管理员使用手册》第四章对应章节，操作逻辑完全一致。

### 4.8 批量导入账户

超级管理员的批量导入功能支持**所有角色**（学生、教师、管理员），而普通管理员只能批量导入学生。

**步骤一：下载导入模板**

**第一步：** 在账户管理页面，点击【批量导入】按钮。

**第二步：** 在弹出的导入模态框中，点击【下载模板】，获取"用户导入模板.csv"文件。

**第三步：** 用 Excel 打开模板，查看列名和示例数据。

---

**步骤二：按模板填写数据**

模板包含以下字段：

| 列名 | 是否必填 | 说明 |
|------|----------|------|
| 用户名 | **必填** | 学号/工号，系统唯一标识 |
| 姓名 | **必填** | 真实姓名 |
| 角色 | **必填** | student / teacher / admin / superadmin |
| 性别 | 必填（推荐） | 男 / 女 |
| 学院 | 学生必填，其他角色可留空 | 必须与系统设置中的学院名称完全一致 |
| 年级 | 学生必填，其他角色可留空 | 必须与系统设置中的年级名称完全一致 |
| 班级 | 学生必填，其他角色可留空 | 必须与系统设置中的班级名称完全一致 |

**第四步：** 按格式填写数据，同一文件中可以包含不同角色的账户（混合导入），最多 1000 行，保存为 CSV 格式。

---

**步骤三：上传并导入**

**第五步：** 在导入模态框中点击【选择文件】，选择填好的 CSV 文件（不超过 5MB）。

**第六步：** 点击【开始导入】，系统开始处理。

**第七步：** 导入完成后，查看系统显示的结果摘要（成功数量 / 失败数量 / 前10条错误信息）。

**第八步：** 根据错误信息修正 CSV 文件中的问题数据，重新导入失败的行。

> ⚠️ **注意：**
> - 角色字段必须使用英文小写：`student`、`teacher`、`admin`、`superadmin`
> - 教师和管理员无需填写学院、年级、班级字段，留空即可
> - 批量导入的账户初始密码均为用户名，需通知相关人员首次登录修改

### 4.9 批量操作说明

在账户列表中勾选多个账户后，可执行以下批量操作：

| 批量操作 | 说明 | 注意事项 |
|----------|------|----------|
| 批量重置密码 | 将选中账户密码全部重置为各自用户名 | 仅支持超级管理员有权管理的角色 |
| 批量毕业归档 | 将选中的学生账户全部执行毕业归档 | 仅限学生账户；不可撤销 |
| 批量删除 | 删除选中的所有账户 | 高危操作，不可撤销 |

**操作步骤（以批量重置密码为例）：**

**第一步：** 通过筛选找到目标账户群体。

**第二步：** 勾选需要操作的账户行复选框（全选可点击表头复选框）。

**第三步：** 点击批量操作工具栏中的对应按钮。

**第四步：** 在确认框中确认操作，点击【确认】执行。

## 第五章 信息审核

信息审核功能与普通管理员完全相同，超级管理员拥有全部审核权限。以下为操作步骤摘要，详细说明请参阅《管理员使用手册》第五章。

### 5.1 进入信息审核页面

点击左侧菜单中的【信息审核】选项（菜单旁的红色数字徽章显示待审核数量），进入审核页面。

页面顶部有三个标签页：**待审核 / 已通过 / 已拒绝**，默认显示"待审核"。

### 5.2 待审核列表说明

待审核列表展示所有等待处理的学生信息修改申请，按申请批次（batch_id）分组显示。

**列表字段：** 复选框、申请人（学号+姓名）、修改字段列表、提交时间、操作按钮（详情/通过/拒绝）。

**实时刷新：** 系统每 5 秒自动从服务器获取最新待审核列表，新申请会以高亮动画出现在列表中，左侧菜单徽章数量同步更新，无需手动刷新页面。

### 5.3 查看审核详情

**第一步：** 在待审核列表中，点击目标申请行右侧的【详情】按钮。

**第二步：** 弹出审核详情弹窗，以表格形式逐字段对比展示修改前后的值：

| 字段名称 | 修改前（原值） | 申请修改为 |
|----------|----------------|-----------|
| 政治面貌 | 入党积极分子 | 中共预备党员 |
| 联系方式 | 138xxxx1234 | 156xxxx5678 |

**第三步：** 核实修改内容是否真实准确后，可在弹窗中直接点击【通过】或【拒绝】，或关闭弹窗在列表中操作。

### 5.4 通过审核申请

**单条通过：**

**第一步：** 在待审核列表中，点击目标申请对应的【通过】按钮（绿色）。

**第二步：** 确认弹窗中点击【确认通过】。

**第三步：** 系统将申请中所有字段的新值更新到学生信息表，申请状态变为"已通过"，记录审核人和时间。

### 5.5 拒绝审核申请

**单条拒绝：**

**第一步：** 点击目标申请对应的【拒绝】按钮（红色）。

**第二步：** 在拒绝弹窗中填写拒绝原因（建议详细填写，帮助学生理解并修正）。

**第三步：** 点击【确认拒绝】，申请状态更新为"已拒绝"，学生可在其待审核列表中看到拒绝原因。

### 5.6 批量通过与批量拒绝

**批量通过：**

**第一步：** 在待审核列表中勾选多条申请的复选框。

**第二步：** 点击批量操作区域的【批量通过】按钮，在确认框中点击【确认】。

**第三步：** 系统批量更新选中申请对应的学生信息，状态全部变为"已通过"。

---

**批量拒绝：**

**第一步：** 勾选需要拒绝的多条申请。

**第二步：** 点击【批量拒绝】按钮，在弹窗中输入统一拒绝原因（可选）。

**第三步：** 点击【确认拒绝】，选中申请状态全部变为"已拒绝"。

### 5.7 历史审核记录查看

点击顶部【已通过】或【已拒绝】标签页，查看历史审核记录，包含审核时间、审核人、申请内容详情和拒绝原因。

**第三步：** 上传填好的文件，点击【开始导入】，系统按学号匹配并更新或插入学生详情数据。

**第四步：** 查看导入结果摘要，处理失败行后重新导入。

## 第八章 系统设置

与普通管理员功能完全相同，超级管理员拥有全部系统设置权限。以下为摘要说明，详细操作请参阅《管理员使用手册》第八章。

### 8.1 进入系统设置页面

点击左侧菜单中的【系统设置】，页面显示五个标签页：**学院 / 年级 / 班级 / 政治面貌 / 发展时间**。

> ⚠️ **重要提示：** 系统设置中的选项是学生填写信息时下拉框的数据来源。修改选项会直接影响学生填写体验，建议谨慎操作，避免在学生集中填写信息期间大幅调整选项。

### 8.2 管理学院选项

切换到【学院】标签页：
- **添加：** 在新增输入框输入学院名称，点击【添加】
- **编辑：** 点击行内【编辑】按钮，修改名称后点击【保存】
- **删除：** 点击行内【删除】按钮，确认后删除

### 8.3 管理年级选项

切换到【年级】标签页，操作同学院（增/改/删），年级名称建议格式统一为"XXXX级"。

### 8.4 管理班级选项

切换到【班级】标签页，操作同学院（增/改/删）。

### 8.5 管理政治面貌选项

切换到【政治面貌】标签页，操作同学院（增/改/删）。

> ⚠️ **注意：** 不要删除"入党积极分子"、"中共预备党员"、"中共党员"这三个核心政治面貌选项，它们是学生政治面貌升级路径的关键节点。

### 8.6 管理发展时间

切换到【发展时间】标签页，发展时间是特殊选项，存储格式为"名称|日期"。

**添加发展时间：**

**第一步：** 在新增区域，找到时间名称和日期两个输入框。

**第二步：** 在时间名称输入框输入描述性名称（如"2025年入党积极分子确认"）。

**第三步：** 在日期输入框选择具体日期（如"2025-03-20"）。

**第四步：** 点击【添加】，新条目出现在列表中，学生填写信息时相关字段将自动读取此日期。

**编辑发展时间：** 点击行内【编辑】，修改名称或日期后点击【保存】。

**删除发展时间：** 点击行内【删除】，确认后删除。

> ⚠️ **注意：** 每年度应及时添加新一年的发展时间配置，以确保当年入学的新生填写信息时，相关日期字段能正确自动填入。

### 8.7 拖拽排序操作步骤

所有选项列表均支持拖拽排序：

**第一步：** 将鼠标移至选项行左侧的拖拽图标（"⋮⋮"或"≡"）上。

**第二步：** 鼠标变为移动光标，按住鼠标左键。

**第三步：** 拖拽至目标位置后松开鼠标。

**第四步：** 系统自动保存新排序，无需手动确认。

## 第九章 反馈管理（超级管理员专属）

本章功能**仅超级管理员可见**，普通管理员的左侧菜单中不会出现"反馈管理"入口。

### 9.1 进入反馈管理页面

**第一步：** 在左侧菜单中，找到【反馈管理】选项（通常位于【操作日志】上方或下方）。

**第二步：** 点击【反馈管理】，进入反馈管理页面。

> ⚠️ **注意：** 如果您在左侧菜单中找不到【反馈管理】入口，请确认您当前登录的是超级管理员账号（角色为 superadmin），而非普通管理员账号（角色为 admin）。

### 9.2 反馈统计卡片说明

反馈管理页面顶部显示四个统计卡片，提供反馈整体情况的快速概览：

| 卡片名称 | 含义说明 |
|----------|----------|
| 总反馈数 | 系统收到的所有用户反馈总数量（含所有状态） |
| 待处理 | 状态为"待处理"的反馈数量（新提交、尚未开始处理） |
| 处理中 | 状态为"处理中"的反馈数量（已回复、正在跟进） |
| 已解决 | 状态为"已解决"的反馈数量（问题已解决或建议已采纳） |

通过这四个数字，可以快速判断当前积压的待处理反馈量，以及处理进度。

### 9.3 筛选反馈

反馈列表支持以下筛选条件，便于聚焦处理特定类型的反馈：

**按反馈类型筛选：**

**第一步：** 找到【反馈类型】筛选下拉框。

**第二步：** 从以下选项中选择：
- **全部类型：** 显示所有反馈（默认）
- **Bug 反馈：** 仅显示用户报告的功能异常或错误
- **功能建议：** 仅显示用户提交的改进建议

---

**按状态筛选：**

**第一步：** 找到【反馈状态】筛选下拉框。

**第二步：** 从以下选项中选择：
- **全部状态：** 显示所有反馈（默认）
- **待处理：** 仅显示尚未处理的反馈
- **处理中：** 仅显示正在处理的反馈
- **已解决：** 仅显示已解决的反馈
- **已关闭：** 仅显示已关闭的反馈

**第三步：** 选择筛选条件后，反馈列表自动更新（或点击【搜索】按钮）。

**第四步：** 点击【重置】恢复显示全部反馈。

### 9.4 查看反馈详情

反馈列表展示每条反馈的概要信息：

| 字段 | 说明 |
|------|------|
| 反馈类型 | Bug反馈 / 功能建议（彩色徽章区分） |
| 提交人 | 提交该反馈的用户姓名和角色 |
| 联系方式 | 用户填写的联系方式（如填写了的话） |
| 反馈标题 | 反馈标题 |
| 提交时间 | 反馈提交的时间 |
| 当前状态 | 待处理 / 处理中 / 已解决 / 已关闭 |
| 操作按钮 | 查看详情 / 回复 / 更新状态 |

**查看 Bug 反馈详情：**

**第一步：** 在反馈列表中，找到类型为"Bug反馈"的条目，点击【查看详情】按钮（或点击标题）。

**第二步：** 弹出详情面板或跳转到详情页，展示该 Bug 反馈的完整信息：

- **基本信息：** 提交人、提交时间、联系方式
- **反馈标题与详细描述：** 用户描述的问题内容
- **Bug 特有信息：**
  - 设备类型（Android / iOS / HarmonyOS / Windows / Mac）
  - 设备型号（如"iPhone 14 Pro"、"HUAWEI Mate 60 Pro"）
  - 问题发生时间（用户选择的日期时间）
  - 截图（若用户上传了截图，此处显示图片）
- **管理员回复：** 若已回复，显示回复内容

**第三步：** 仔细阅读用户描述和查看截图，理解问题的具体情况。

**第四步：** 查看完毕后关闭详情面板，根据情况决定回复内容或处理状态。

**查看功能建议详情：**

**第二步：** 弹出详情面板，展示：
- 提交人、提交时间、联系方式
- 建议标题与详细描述
- 管理员回复（若已有）

### 9.5 回复用户反馈

处理反馈时，建议向用户回复处理进度或结论，让用户知晓其反馈已被关注：

**第一步：** 在反馈列表中，找到需要回复的反馈条目。

**第二步：** 点击该条目右侧的【回复】按钮。

**第三步：** 系统弹出回复输入框（或展开回复表单）。

**第四步：** 在回复输入框中输入您的回复内容，例如：
- 针对 Bug 反馈：`"您好，感谢您的反馈！该问题已确认，正在修复中，预计本周内完成更新。"`
- 针对功能建议：`"您的建议很有价值，我们已记录并纳入下一版本的功能规划中，感谢！"`

**第五步：** 点击【提交回复】按钮。

**第六步：** 系统保存回复内容到数据库（`feedback.admin_reply` 字段），同时自动将该反馈状态更新为"**处理中**"（如果原状态为"待处理"）。

**第七步：** 提交回复后，该用户下次查看反馈页面时，即可看到您的回复内容。

> ⚠️ **注意：** 
> - 提交回复后，反馈状态自动从"待处理"变为"处理中"，表示您已开始跟进
> - 回复内容可多次更新（再次点击【回复】即可覆盖之前的回复）
> - 回复语气建议保持专业、友好，感谢用户的反馈

### 9.6 更新反馈状态

除了通过回复自动变更状态外，您也可以手动更新反馈状态：

**第一步：** 在反馈列表中，找到需要更新状态的条目。

**第二步：** 点击该条目右侧的【更新状态】按钮（或在详情面板中找到状态选择器）。

**第三步：** 在状态下拉框中，从以下选项中选择新状态：
- **处理中：** 正在处理，已与用户沟通或正在修复 Bug
- **已解决：** Bug 已修复，或建议已采纳实现；问题完全解决
- **已关闭：** 无法复现、不予采纳、重复反馈、或其他原因不再处理

**第四步：** 点击【确认更新】按钮，系统更新该反馈的状态。

**第五步：** 反馈列表中该条目的状态徽章颜色随之改变，反映最新状态。

> ⚠️ **注意：** 建议在将状态更新为"已解决"或"已关闭"之前，先回复用户，说明处理结果，以便用户了解进展。

### 9.7 反馈状态流转说明

反馈的状态按以下规则流转：

```
用户提交 → 待处理（pending）
                ↓ 管理员回复 / 手动更新
            处理中（processing）
                ↓ 问题修复 / 建议采纳
            已解决（resolved）
                ↓ 无效/重复/不采纳
            已关闭（closed）
```

**各状态的显示颜色：**

| 状态 | 徽章颜色 | 适用场景 |
|------|----------|----------|
| 待处理（pending） | 黄色 | 刚提交、尚未查看处理 |
| 处理中（processing） | 蓝色 | 已回复用户、正在跟进修复 |
| 已解决（resolved） | 绿色 | Bug 已修复或建议已实现 |
| 已关闭（closed） | 灰色 | 反馈无效、重复或不予处理 |

**建议的处理流程：**

1. **新反馈（待处理）：** 每天查看反馈管理页面，确认是否有新的待处理反馈
2. **确认问题：** 对 Bug 反馈，尝试按用户描述复现问题；对功能建议，评估可行性
3. **回复用户：** 无论是否能立即解决，都建议先回复用户告知已收到反馈
4. **处理完成：** 问题修复后将状态改为"已解决"；无法处理的改为"已关闭"并说明原因

## 第十二章 常见问题 FAQ

### 12.1 如何为新老师批量创建账户

**场景：** 每学期开始，需要为多名新任课教师或新进辅导员创建系统账户。

**操作步骤：**

**第一步：** 整理所有新教师的信息，包括：姓名、工号（作为用户名）、性别。

**第二步：** 下载账户导入模板（账户管理页 → 批量导入 → 下载模板）。

**第三步：** 按模板格式填写教师信息，角色列填写 `teacher`，学院/年级/班级列留空，保存为 CSV 文件：

```
用户名,姓名,角色,性别
T2025001,李老师,teacher,女
T2025002,王老师,teacher,男
```

**第四步：** 上传 CSV 文件，点击【开始导入】，系统批量创建教师账户。

**第五步：** 导入成功后，将各教师的用户名（工号）和初始密码（与用户名相同）以安全方式（如工作邮件、当面告知）分别通知相关教师。

**第六步：** 提醒教师首次登录时必须修改密码，并建议绑定邮箱以便后续找回密码。

### 12.2 如何处理已离职教师的账户

**场景：** 某位教师离职或调岗，需要处理其系统账户。

**推荐处理方式（禁用而非删除）：**

**第一步：** 进入账户管理页面，在角色筛选中选择"教师"，搜索该教师姓名或工号。

**第二步：** 找到该教师账户，点击【禁用】按钮，在确认框中点击【确认】。

**第三步：** 账户状态变为"禁用"，该教师账号立即无法登录，但历史操作记录保留在日志中。

> ⚠️ **为什么建议禁用而非删除：**
> - 保留账户可以保留该教师的历史登录和操作记录，便于后续审计
> - 若未来需要恢复使用，直接点击【启用】即可，无需重新创建
> - 删除账户后，相关日志中的账户关联信息将丢失

---

**若必须彻底删除账户（如数据合规要求）：**

**第一步：** 确认该教师账户下无重要的历史操作记录需要保留（通过操作日志查询确认）。

**第二步：** 在账户列表中点击【删除】按钮，在二次确认框中点击【确认删除】。

### 12.3 反馈截图无法显示怎么办

**问题：** 在反馈管理详情中，用户上传的截图显示为"图片无法加载"或空白。

**可能原因及解决方法：**

1. **文件路径问题：** 截图保存在服务器的 `uploads/` 目录下，若该目录权限设置错误，图片无法被 Web 服务器访问
   - 解决：联系服务器管理员检查 `uploads/` 目录的权限设置，确保 Apache/Nginx 有读取权限

2. **文件已被删除：** 若服务器清理了 `uploads/` 目录中的文件，图片链接失效
   - 解决：此情况下图片无法恢复，可请用户重新提交带截图的反馈

3. **浏览器缓存问题：** 按 `Ctrl + F5` 强制刷新页面后重试

4. **文件格式不支持：** 若用户上传了非图片格式的文件被误命名为图片，会导致无法显示
   - 解决：查看数据库中该反馈记录的 `screenshot` 字段值，确认文件名和扩展名

### 12.4 超级管理员账号密码忘记怎么办

**问题：** 超级管理员忘记了登录密码，且账号未绑定邮箱，无法通过系统找回。

**解决方案：需要直接操作服务器数据库。**

> ⚠️ **警告：** 以下操作需要直接访问服务器和数据库，请确保操作者具备相应的技术能力，或委托系统部署人员操作。不当的数据库操作可能导致数据损坏。

**操作步骤（需服务器管理员执行）：**

**第一步：** 通过 SSH 或远程桌面连接到服务器。

**第二步：** 登录 MySQL 数据库：
```bash
mysql -u root -p
```
输入数据库 root 密码。

**第三步：** 切换到系统数据库：
```sql
USE shxyinfo;
```

**第四步：** 生成新密码的 bcrypt 哈希值（需要在 PHP 环境中执行）：
```php
# 在服务器上运行 PHP 命令生成哈希
php -r "echo password_hash('新密码', PASSWORD_DEFAULT);"
```
记录输出的哈希字符串（格式类似 `$2y$10$...`）。

**第五步：** 在 MySQL 中更新超级管理员密码：
```sql
UPDATE users 
SET password = '上一步生成的哈希字符串', force_change_password = 1
WHERE role = 'superadmin' AND username = '超级管理员用户名';
```

**第六步：** 验证更新成功：
```sql
SELECT id, username, role FROM users WHERE role = 'superadmin';
```

**第七步：** 退出 MySQL，使用新密码登录系统，登录后立即修改密码并绑定邮箱。

> ⚠️ **预防措施：** 为避免此情况发生，强烈建议：
> 1. 登录系统后立即绑定个人邮箱
> 2. 将密码记录在安全的密码管理工具中（如 KeePass、1Password 等）
> 3. 定期测试密码是否正确，避免因长期不用而遗忘

### 12.5 如何新建下一年度的管理员账户

**场景：** 新学年开始，需要为新一届班委或党支部负责人创建管理员账户。

**操作步骤：**

**第一步：** 进入账户管理页面，点击【添加账户】。

**第二步：** 在模态框中填写：
- 用户名：为新管理员设定唯一的账号名（如工号或自定义账号）
- 姓名：管理员真实姓名
- 角色：选择【管理员】
- 性别：男/女
- 初始密码：设置一个初始密码（建议比默认"用户名=密码"更复杂）

**第三步：** 点击【提交】创建账户。

**第四步：** 以安全方式将账号信息告知新管理员，并要求其首次登录后立即修改密码。

**第五步：** 对于不再担任管理员职务的旧管理员账户，执行【禁用】操作（或删除）。

**批量新增管理员（如需同时新增多个）：**

参照 4.8 节"批量导入账户"，在 CSV 中将角色列设为 `admin`，一次性导入多个管理员账户。
