## 第一章 系统简介

### 1.1 系统概述与管理员职责

生化学院党员信息管理系统的管理员是系统日常运维和数据管理的核心角色。管理员负责：

- **账户管理：** 为学生创建账号、重置密码、启用/禁用账号、处理学籍变动（毕业归档、转班）
- **信息审核：** 审核学生提交的信息修改申请，决定通过或拒绝
- **数据管理：** 查看和导出在校学生及毕业生数据、批量导入学生信息
- **系统设置：** 维护系统中的下拉选项（学院、年级、班级、政治面貌、发展时间）
- **日志查看：** 查看系统操作日志，掌握系统使用情况
- **数据统计：** 通过数据展板掌握全院党员发展整体情况

### 1.2 管理员与超级管理员的区别

系统中存在两个管理级别，普通管理员与超级管理员的主要区别如下：

| 功能 | 普通管理员（admin） | 超级管理员（superadmin） |
|------|---------------------|--------------------------|
| 创建学生账户 | ✅ 可以 | ✅ 可以 |
| 创建教师账户 | ❌ 不可以 | ✅ 可以 |
| 创建管理员账户 | ❌ 不可以 | ✅ 可以 |
| 管理学生账户（重置密码/禁用/删除） | ✅ 可以 | ✅ 可以 |
| 管理教师/管理员账户 | ❌ 不可以 | ✅ 可以 |
| 信息审核 | ✅ 可以 | ✅ 可以 |
| 学生信息管理 | ✅ 可以 | ✅ 可以 |
| 系统设置 | ✅ 可以 | ✅ 可以 |
| 操作日志查看 | ✅ 可以 | ✅ 可以 |
| 反馈管理 | ❌ 不可以 | ✅ 可以 |
| 数据展板（角色人数分布） | ❌ 不显示 | ✅ 显示 |

本手册面向普通管理员编写，超级管理员功能请参阅《超级管理员使用手册》。

### 1.3 浏览器与设备要求

| 浏览器 | 推荐版本 |
|--------|----------|
| Google Chrome | 90 及以上 |
| Microsoft Edge | 90 及以上 |
| Firefox | 88 及以上 |
| Safari | 14 及以上 |

建议在电脑端使用，以便进行批量操作和数据导入导出。

## 第二章 账号与登录

### 2.1 首次登录与强制修改密码

**第一步：** 打开浏览器，访问系统登录页面。

**第二步：** 在【角色】下拉菜单中选择【管理员】。

**第三步：** 在【用户名】输入框中输入您的管理员用户名（由超级管理员提供）。

**第四步：** 在【密码】输入框中输入初始密码（通常与用户名相同，或由超级管理员单独告知）。

**第五步：** 点击【登录】按钮。

**第六步：** 系统检测到首次登录，自动跳转至【修改密码】页面。

**第七步：** 在【当前密码】输入框中输入初始密码。

**第八步：** 在【新密码】输入框中输入您要设置的新密码（建议包含大小写字母和数字，长度不低于8位）。

**第九步：** 在【确认新密码】输入框中再次输入相同新密码。

**第十步：** 点击【修改密码】按钮，修改成功后自动跳转至数据展板。

> ⚠️ **注意：** 管理员账号的安全至关重要，请设置复杂度高的密码，并妥善保管，不要告知他人。

### 2.2 日常登录步骤

**第一步：** 访问系统登录页面，在【角色】下拉框中选择【管理员】。

**第二步：** 输入用户名和密码，点击【登录】按钮，登录成功后跳转至数据展板。

### 2.3 忘记密码与重置密码

**前提：** 账号已绑定邮箱。若未绑定邮箱，请联系超级管理员重置密码。

**第一步：** 在登录页面点击【忘记密码？】链接。

**第二步：** 在忘记密码页面输入账号绑定的邮箱地址。

**第三步：** 点击【发送验证码】，查收邮件中的 6 位验证码（有效期 10 分钟）。

**第四步：** 输入验证码，点击【验证】。

**第五步：** 验证通过后，设置新密码并确认，点击【重置密码】，重置成功后重新登录。

### 2.4 退出登录

点击左侧菜单底部或右上角的【退出登录】按钮，系统清除登录状态，返回登录页面。

## 第三章 数据展板

### 3.1 进入数据展板

登录后系统默认进入数据展板。也可随时点击左侧菜单顶部的【数据展板】进入。

### 3.2 数据概览卡片说明

页面顶部显示以下统计卡片：

| 卡片名称 | 含义说明 |
|----------|----------|
| 在校学生总数 | 当前状态正常的在校学生总人数（不含已毕业） |
| 正式党员 | 政治面貌为"中共党员"/"正式党员"的学生人数 |
| 预备党员 | 政治面貌为"中共预备党员"/"预备党员"的学生人数 |
| 入党积极分子 | 政治面貌为"入党积极分子"的学生人数 |
| 待审核申请 | 当前等待审核的学生信息修改申请数量 |
| 毕业生档案 | 已归档的毕业生总人数 |
| 系统账户总数 | 系统中所有角色账户的总数量（含学生、教师、管理员） |

> ⚠️ **说明：** 统计数字约每 5 秒自动刷新，始终显示最新数据。

### 3.3 筛选功能使用

**第一步：** 在展板顶部找到筛选下拉框：年级、班级、政治面貌、性别。

**第二步：** 点击需要筛选的下拉框，从列表中选择条件。

**第三步：** 页面自动刷新，统计卡片和图表均按所选条件重新计算。

**第四步：** 将所有下拉框重置为"全部"即可取消筛选。

### 3.4 图表说明

数据展板包含 6 个统计图表（向下滚动页面可见）：

| 图表 | 类型 | 内容说明 |
|------|------|----------|
| 政治面貌分布 | 甜甜圈图 | 各政治面貌的人数占比，鼠标悬停查看精确数字 |
| 各年级人数 | 柱状图 | 各届年级的学生人数对比 |
| 各班级人数 | 柱状图 | 各班级的学生人数对比 |
| 性别比例 | 饼图 | 男女生人数及占比 |
| 民族分布 Top 10 | 水平条形图 | 人数最多的前10个民族 |
| 历年党员发展趋势 | 折线图 | 近年来每年新增预备党员的变化趋势 |

所有图表均支持鼠标悬停查看精确数字，并随顶部筛选条件同步更新。

### 3.5 实时数据更新说明

系统每隔约 5 秒自动向服务器请求最新数据，以下内容会自动刷新：
- 顶部统计卡片数字（含"待审核申请"数量）
- 左侧菜单【信息审核】旁的红色数字徽章（待审核数量）

当数字更新时会有短暂的动画效果提示您数据已刷新。**无需手动刷新页面**。

## 第四章 账户管理

### 4.1 进入账户管理页面

**第一步：** 点击左侧菜单中的【账户管理】选项。

**第二步：** 页面显示所有由管理员可管理的账户列表（普通管理员只能看到学生账户）。

### 4.2 搜索与筛选账户

账户列表支持以下方式缩小范围：

**关键词搜索：**
- 在搜索框中输入用户名（学号）或姓名，点击【搜索】或按回车键
- 支持模糊搜索（如输入"2022"可找到所有2022级学生）

**条件筛选：**

| 筛选条件 | 说明 |
|----------|------|
| 角色 | 学生 / 教师 / 管理员（普通管理员只显示学生选项） |
| 账户状态 | 启用 / 禁用 / 全部 |
| 年级 | 按年级筛选学生账户 |
| 信息完善状态 | 已完善 / 未完善（是否已填写个人信息） |

**第一步：** 填写搜索框或选择筛选条件。

**第二步：** 点击【搜索】/【查询】按钮，列表自动更新。

**第三步：** 点击【重置】清除所有条件，恢复显示全部账户。

**每页显示数量：** 可在列表右上角切换每页显示 20条 / 50条 / 全部。

### 4.3 新增单个账户

**第一步：** 在账户管理页面，点击右上角的【添加账户】按钮。

**第二步：** 页面弹出"新增账户"模态框，按以下顺序填写信息：

**第三步：** 在【用户名】输入框中输入学生的学号（用户名唯一，不可重复）。

**第四步：** 在【姓名】输入框中输入学生的真实姓名。

**第五步：** 在【角色】下拉框中选择【学生】（普通管理员只能创建学生账户）。

**第六步：** 角色选择学生后，页面出现学生专属字段，依次填写：
- **性别：** 点击下拉框，选择"男"或"女"
- **学院：** 点击下拉框，选择所属学院（来自系统设置中的学院选项）
- **年级：** 点击下拉框，选择入学年级（来自系统设置中的年级选项）
- **班级：** 点击下拉框，选择所属班级（来自系统设置中的班级选项）

**第七步：**（可选）在【初始密码】输入框中设置该账户的初始密码。若不填写，初始密码默认为用户名（即学号），且系统会标记该账户"下次登录必须修改密码"。

**第八步：** 确认所有信息填写正确后，点击【提交】（或【确认添加】）按钮。

**第九步：** 系统创建账户，提示"账户创建成功"，模态框关闭，列表自动刷新，新账户出现在列表中。

> ⚠️ **注意：**
> - 用户名（学号）一旦创建不可修改，请仔细核对
> - 账户创建后，学生登录时系统将强制要求其修改初始密码
> - 系统同时为该学生创建基础的个人信息记录（仅包含管理员填写的学号、姓名、性别、学院、年级、班级），其余信息需学生自行登录填写

### 4.4 批量导入账户

当需要批量创建多个学生账户时，可使用 CSV 文件批量导入功能。

**步骤一：下载导入模板**

**第一步：** 在账户管理页面，点击【批量导入】按钮（或【导入账户】按钮）。

**第二步：** 弹出导入模态框，点击其中的【下载模板】按钮（或链接）。

**第三步：** 浏览器下载名为"用户导入模板.csv"的模板文件。

**第四步：** 用 Excel 或 WPS 打开模板文件，查看表头（列名）和示例数据。

---

**步骤二：按模板填写数据**

模板文件包含以下必填列：

| 列名 | 说明 | 示例 |
|------|------|------|
| 用户名 | 学号，系统唯一标识 | 20220101 |
| 姓名 | 学生真实姓名 | 张三 |
| 角色 | 固定填写"student" | student |
| 性别 | "男"或"女" | 男 |
| 学院 | 必须与系统设置中的学院名称完全一致 | 生命科学与化学学院 |
| 年级 | 必须与系统设置中的年级名称完全一致 | 2022级 |
| 班级 | 必须与系统设置中的班级名称完全一致 | 生物1班 |

**第五步：** 按以上格式，在模板中填写全部学生数据（每行一名学生，最多 1000 行）。

**第六步：** 保存文件，注意保存为 **CSV 格式**（UTF-8 编码或 GBK 编码均可，系统会自动识别）。

---

**步骤三：上传文件并导入**

**第七步：** 返回系统导入模态框，点击【选择文件】按钮（或将文件拖拽至上传区域），选择填写好的 CSV 文件。

**第八步：** 点击【开始导入】按钮，系统开始处理文件。

**第九步：** 导入完成后，系统显示结果摘要：
- **成功导入：** N 条（已成功创建的账户数量）
- **失败：** N 条（创建失败的数量，并列出前 10 条错误原因）

**第十步：** 查看失败原因（如"用户名已存在"、"班级不存在"等），修正对应行数据后可重新导入失败的部分。

> ⚠️ **注意：**
> - 文件大小不超过 5MB，最多 1000 行数据
> - 学院、年级、班级的名称必须与系统设置中完全一致（包括全角/半角、空格），否则会报错
> - 如果用户名已存在（学号已有账号），系统会更新该学生的基础信息（学院、年级、班级等），不会报错也不会重复创建
> - 导入的新账户初始密码均为用户名（学号），首次登录时会强制修改密码

### 4.5 重置账户密码

当学生忘记密码且没有绑定邮箱时，管理员可代为重置密码：

**第一步：** 在账户管理页面，找到需要重置密码的学生账户所在行。

**第二步：** 点击该行操作列中的【重置密码】按钮。

**第三步：** 系统弹出确认框，提示"请输入您（管理员自己）的登录密码以确认此操作"。

**第四步：** 在确认框的密码输入框中，输入**您自己（管理员）的当前登录密码**（用于安全验证，防止误操作）。

**第五步：** 点击【确认重置】按钮。

**第六步：** 系统将目标学生的密码重置为其**用户名（学号）**，同时标记该账户"下次登录必须修改密码"（`force_change_password=1`）。

**第七步：** 系统提示"密码重置成功"。通知该学生使用学号作为密码重新登录，登录后系统会强制要求其设置新密码。

> ⚠️ **注意：**
> - 重置密码需要输入**管理员自己的密码**进行二次验证，这是防止误操作的安全机制
> - 重置后的密码为该学生的学号，请告知学生尽快重新登录修改密码
> - 此操作会被记录在操作日志中

### 4.6 启用与禁用账户

**禁用账户（阻止登录）：**

**第一步：** 在账户列表中，找到需要禁用的账户所在行，该账户状态徽章显示为绿色"启用"。

**第二步：** 点击该行操作列中的【禁用】按钮（或状态切换按钮）。

**第三步：** 系统弹出确认框，点击【确认】。

**第四步：** 账户状态变更为红色"禁用"，该学生账号立即无法登录系统（若该学生已登录，其当前会话会在下次操作时失效）。

---

**启用账户（恢复登录）：**

**第一步：** 在账户列表中，找到状态为红色"禁用"的账户所在行。

**第二步：** 点击该行操作列中的【启用】按钮。

**第三步：** 系统弹出确认框，点击【确认】。

**第四步：** 账户状态恢复为绿色"启用"，该学生可以正常登录系统。

> ⚠️ **注意：** 已毕业学生（is_graduated=1）的账户即使被启用，也无法正常登录，因为系统对毕业状态有额外限制。

### 4.7 学籍变动——单个学生毕业归档

当某位学生毕业时，需执行毕业归档操作，将其信息存入毕业生库并禁用账号：

**第一步：** 在账户管理页面，找到需要归档的学生账户所在行。

**第二步：** 点击该行操作列中的【学籍变动】按钮。

**第三步：** 系统弹出"学籍变动"模态框，其中有两个操作选项。

**第四步：** 选择【设为毕业】选项（点击对应单选按钮或按钮）。

**第五步：** 系统提示确认信息："确定将该学生标记为毕业并归档吗？此操作完成后，该学生账号将被禁用，且不可撤销。"

**第六步：** 确认无误后，点击【确认执行】按钮。

**第七步：** 系统执行以下操作（自动完成）：
1. 将该学生的完整个人信息复制到毕业生库（`graduated_students` 表），记录归档时间
2. 删除该学生在在校学生信息表中的记录
3. 删除该学生所有处于"待审核"状态的信息修改申请
4. 将该学生账号标记为已毕业（`is_graduated=1`），并禁用账号（`is_active=0`）

**第八步：** 系统提示"毕业归档成功"。该学生信息可在【毕业生管理】页面查找，其账号无法再登录系统。

> ⚠️ **注意：**
> - 毕业归档操作**不可撤销**，请务必确认学生信息无误后再执行
> - 归档前请确认该学生的个人信息已全部填写完整，否则归档后数据将不完整
> - 执行后操作会记录在操作日志中

### 4.8 学籍变动——转专业或班级

当学生发生转专业或转班情况时，可通过此功能更新其所在班级：

**第一步：** 在账户管理页面，找到需要转班的学生账户所在行。

**第二步：** 点击该行操作列中的【学籍变动】按钮。

**第三步：** 在学籍变动模态框中，选择【转专业/班级】选项。

**第四步：** 页面显示当前学生的学院、年级、班级信息，以及需要修改的下拉框。

**第五步：** 根据实际情况，修改以下字段（可只修改其中一项或多项）：
- **学院：** 点击下拉框，选择新的学院
- **年级：** 点击下拉框，选择新的年级
- **班级：** 点击下拉框，选择新的班级

**第六步：** 确认填写正确后，点击【确认执行】按钮。

**第七步：** 系统更新该学生在 `student_info` 表中的学院、年级、班级字段，提示"学籍变动成功"。

> ⚠️ **注意：** 转班操作会直接修改学生信息，不需要学生提交申请，也不进入审核流程。操作会被记录在日志中。

### 4.9 删除账户

> ⚠️ **警告：** 删除账户是**不可逆操作**，会同时删除该账户关联的所有学生信息和历史申请记录，请谨慎操作。

**第一步：** 在账户管理页面，找到需要删除的账户所在行。

**第二步：** 点击该行操作列中的【删除】按钮。

**第三步：** 系统弹出二次确认对话框："确定要删除该账户吗？此操作不可撤销，将同时删除该账户的所有个人信息。"

**第四步：** 仔细确认后，点击【确认删除】按钮。

**第五步：** 系统删除 `users` 表中该用户的记录，同时删除关联的 `student_info` 记录和所有 `info_change_requests` 记录。

**第六步：** 系统提示"账户已删除"，列表自动刷新，该账户消失。

> ⚠️ **注意：**
> - 若该学生已提交过审核申请，删除账户时这些记录也会一并删除，无法恢复
> - 如果只是想阻止学生登录，建议使用【禁用账户】而非删除
> - 已毕业归档的学生数据保存在毕业生库中，删除学生账户不影响毕业生库中的数据

### 4.10 批量重置密码

当需要同时重置多个学生的密码时（如新生入学批量创建账号后），可使用批量重置功能：

**第一步：** 在账户管理页面，通过搜索或筛选找到目标学生群体。

**第二步：** 在列表中，勾选需要重置密码的学生行左侧的复选框（可勾选多个）。

**第三步：** 勾选后，页面顶部（或底部）出现批量操作工具栏，点击【批量重置密码】按钮。

**第四步：** 系统弹出确认框，提示"确定要重置已选中 N 个账户的密码吗？重置后密码将变为各账户的用户名。"

**第五步：** 点击【确认】按钮。

**第六步：** 系统批量将选中账户的密码重置为各自的用户名（学号），并设置"下次登录必须修改密码"标记。

**第七步：** 系统提示"批量重置成功，共重置 N 个账户"。

### 4.11 批量毕业归档

学期末时，通常需要对同一届毕业年级的学生批量执行毕业归档：

**第一步：** 在账户管理页面，通过【年级】筛选条件，选择需要毕业归档的年级（如"2020级"）。

**第二步：** 确认筛选结果为目标学生列表，检查列表中是否有遗漏或错误。

**第三步：** 点击列表标题行左侧的全选复选框，选中当前页全部学生；若学生数量超过一页，切换每页显示为"全部"后再全选，或分页批量操作。

**第四步：** 点击批量操作工具栏中的【批量毕业归档】按钮。

**第五步：** 系统弹出确认框："确定要将选中的 N 名学生全部归档为毕业生吗？此操作不可撤销，归档后这些学生将无法登录系统。"

**第六步：** 仔细确认学生名单无误后，点击【确认执行】按钮。

**第七步：** 系统批量执行毕业归档（每位学生的处理逻辑与单个毕业归档相同）。

**第八步：** 系统提示"批量毕业归档成功，共归档 N 名学生"。

> ⚠️ **注意：**
> - 批量毕业归档不可撤销，请确认选中名单准确无误后再执行
> - 归档前建议先导出这批学生的信息作为备份

### 4.12 批量删除账户

**第一步：** 勾选需要删除的多个账户。

**第二步：** 点击批量操作工具栏中的【批量删除】按钮。

**第三步：** 系统弹出二次确认对话框，显示将被删除的账户数量。

**第四步：** 点击【确认删除】，系统批量删除选中账户及其所有关联数据。

> ⚠️ **警告：** 批量删除是高风险操作，请确认后再执行，此操作不可撤销。

## 第五章 信息审核

### 5.1 进入信息审核页面

**方法一：** 点击左侧菜单中的【信息审核】选项。

**方法二：** 当【信息审核】菜单旁有红色数字徽章时（表示有待审核申请），点击该菜单项直接进入。

页面显示信息审核列表，顶部有三个标签页：**待审核 / 已通过 / 已拒绝**，默认显示"待审核"。

### 5.2 待审核列表说明

待审核列表展示所有学生提交的、尚未被处理的信息修改申请：

**列表字段说明：**

| 字段 | 说明 |
|------|------|
| 复选框 | 用于批量操作的勾选框 |
| 申请人 | 提交申请的学生姓名和学号 |
| 申请批次 | 本次申请的批次编号（同一次提交的多个字段修改归为一个批次） |
| 修改字段 | 本批次申请修改的字段名称列表（如"民族、联系方式"） |
| 提交时间 | 学生提交申请的时间 |
| 操作 | 详情 / 通过 / 拒绝 按钮 |

**实时刷新：** 待审核列表每 5 秒自动向服务器查询最新数据，新提交的申请会以高亮动画出现在列表中，无需手动刷新页面。

### 5.3 查看审核详情

在通过或拒绝申请前，建议先查看详情以了解学生具体修改了什么内容：

**第一步：** 在待审核列表中，找到目标申请所在行。

**第二步：** 点击该行右侧的【详情】按钮。

**第三步：** 系统弹出"审核详情"模态框，以表格形式逐字段对比展示：

| 字段名称 | 修改前（原值） | 修改后（申请改为） |
|----------|----------------|-------------------|
| 民族 | 汉族 | 壮族 |
| 联系方式 | 138xxxx1234 | 139xxxx5678 |
| （其他修改字段...） | ... | ... |

**第四步：** 对比修改前后的值，判断修改是否合理准确。

**第五步：** 查看完毕后，可直接在弹窗中点击【通过】或【拒绝】按钮，或关闭弹窗后在列表中操作。

### 5.4 通过单条审核申请

确认申请内容无误后，按以下步骤通过审核：

**第一步：** 在待审核列表中（或审核详情弹窗中），找到目标申请。

**第二步：** 点击该申请对应的【通过】按钮（通常显示为绿色）。

**第三步：** 系统弹出确认提示："确定通过该申请吗？通过后学生信息将立即更新。"

**第四步：** 点击【确认通过】按钮。

**第五步：** 系统执行以下操作：
1. 将该批次申请中每个字段的新值更新到 `student_info` 表中
2. 将申请状态更新为"已通过"（approved）
3. 记录审核人（当前管理员）和审核时间

**第六步：** 系统提示"审核通过，学生信息已更新"，该申请从待审核列表中消失，转移到"已通过"标签页。

### 5.5 拒绝审核申请

如果申请内容有误或不符合规范，按以下步骤拒绝：

**第一步：** 在待审核列表中，找到目标申请。

**第二步：** 点击该申请对应的【拒绝】按钮（通常显示为红色）。

**第三步：** 系统弹出"拒绝申请"模态框，其中包含一个【拒绝原因】文本输入框。

**第四步：**（可选但建议填写）在拒绝原因输入框中，填写拒绝的具体原因（如"身份证号与档案不符，请核实后重新提交"），以便学生了解问题所在并修正后重新提交。

**第五步：** 点击【确认拒绝】按钮。

**第六步：** 系统将申请状态更新为"已拒绝"（rejected），记录拒绝原因、审核人和时间。

**第七步：** 该申请从待审核列表转移到"已拒绝"标签页。学生可在其【待审核列表】页面看到拒绝状态和拒绝原因。

> ⚠️ **注意：** 建议填写清晰的拒绝原因，帮助学生理解问题并及时修正，减少反复沟通。

### 5.6 批量通过审核

当有多条申请需要通过时，可批量操作：

**第一步：** 在待审核列表中，勾选多条需要通过的申请（每行左侧有复选框）。

**第二步：** 点击列表顶部的批量操作区域中的【批量通过】按钮。

**第三步：** 系统弹出确认框："确定通过已选中的 N 条申请吗？"

**第四步：** 点击【确认】，系统批量更新所有选中申请对应的学生信息，状态更新为"已通过"。

**第五步：** 系统提示"批量审核通过，共通过 N 条申请"。

> ⚠️ **注意：** 批量通过前，建议先逐一查看详情确认，特别是涉及重要字段（如政治面貌、党员时间）的修改申请。

### 5.7 批量拒绝审核

**第一步：** 勾选多条需要拒绝的申请。

**第二步：** 点击批量操作区域中的【批量拒绝】按钮。

**第三步：** 系统弹出"批量拒绝"模态框，包含拒绝原因输入框（此原因将统一应用于所有选中申请）。

**第四步：**（可选）输入统一的拒绝原因。

**第五步：** 点击【确认拒绝】，系统批量将选中申请状态更新为"已拒绝"。

### 5.8 已通过与已拒绝历史记录查看

**第一步：** 在信息审核页面，点击顶部的【已通过】或【已拒绝】标签页。

**第二步：** 列表显示历史审核记录，包含审核时间、审核人、申请人、修改字段、审核结果等信息。

**第三步：** 点击【详情】按钮，可查看该条历史记录的字段修改对比（原值 → 新值）以及拒绝原因（已拒绝记录）。

## 第六章 学生信息管理

### 6.1 进入学生信息管理页面

**第一步：** 点击左侧菜单中的【学生信息】选项。

**第二步：** 页面显示所有在校学生的信息列表，默认按学号排序，每页显示15条。

### 6.2 搜索与筛选学生

与教师端相同，支持关键词搜索和多条件筛选（年级、班级、政治面貌、性别）。

**操作步骤：**

**第一步：** 在搜索框中输入学号或姓名（支持模糊搜索）。

**第二步：** 在筛选下拉框中选择年级、班级、政治面貌、性别等条件（可组合使用）。

**第三步：** 点击【搜索】按钮，列表更新为符合条件的学生。

**第四步：** 点击【重置】清除所有筛选条件。

### 6.3 查看学生详细信息

**第一步：** 在列表中，点击目标学生行右侧的【详情】按钮。

**第二步：** 弹出学生详情弹窗，展示该学生的全部信息（包含基本信息、联系信息、身份信息、党员发展信息全部字段）。

**第三步：** 查看完毕后，点击弹窗右上角的【×】关闭。

### 6.4 导出学生信息

**导出全部（当前筛选条件下）：**

**第一步：** 设置好筛选条件（如选择特定年级或政治面貌）。

**第二步：** 点击页面右上角的【导出全部】按钮。

**第三步：** 浏览器下载 CSV 格式文件（文件名含时间戳），文件包含当前筛选条件下全部学生的所有字段信息。

---

**导出选中（部分学生）：**

**第一步：** 在列表中勾选需要导出的学生行左侧复选框（可多选）。

**第二步：** 出现【导出选中（N条）】按钮后，点击该按钮。

**第三步：** 浏览器下载仅包含选中学生信息的 CSV 文件。

> ⚠️ **注意：** 导出文件包含学生身份证号、手机号等敏感信息，请妥善保管，不要随意传播。

### 6.5 批量导入学生信息

当需要批量补录或更新学生详细信息时（如新生入学后批量导入已收集的信息），可使用此功能：

**步骤一：下载学生信息导入模板**

**第一步：** 在学生信息管理页面，点击【批量导入】按钮（仅管理员可见，教师端无此按钮）。

**第二步：** 弹出导入模态框，点击【下载模板】，获取"学生信息导入模板.csv"文件。

**第三步：** 用 Excel 打开模板，查看所有可导入的字段列名。

---

**步骤二：按模板填写学生信息**

模板包含以下字段（学号为必填，其余有则填）：

| 列名 | 是否必填 | 说明 |
|------|----------|------|
| 学号 | **必填** | 用于匹配已有账户 |
| 民族 | 选填 | 必须与系统民族选项一致 |
| 身份证号 | 选填 | 18位 |
| 联系方式 | 选填 | 11位手机号 |
| 邮箱 | 选填 | 有效邮箱格式 |
| 家庭住址 | 选填 | 详细地址 |
| 政治面貌 | 选填 | 必须与系统政治面貌选项一致 |
| 预计毕业时间 | 选填 | 格式：YYYY 或 YYYY-MM |
| 入团时间 | 选填 | 格式：YYYY-MM-DD |
| 递交入党申请书时间 | 选填 | 格式：YYYY-MM-DD |
| 确定积极分子时间 | 选填 | 格式：YYYY-MM-DD |
| 确定预备党员时间 | 选填 | 格式：YYYY-MM-DD |
| 转正时间 | 选填 | 格式：YYYY-MM-DD |

**第四步：** 按模板填写数据（最多 1000 行），保存为 CSV 格式。

---

**步骤三：上传并导入**

**第五步：** 在导入模态框中，点击【选择文件】，选择填好的 CSV 文件。

**第六步：** 点击【开始导入】。

**第七步：** 系统逐行处理：
- 根据学号查找已有账户；若找不到对应账户，该行报错跳过
- 若该学生已有 `student_info` 记录：更新 CSV 中填写的非空字段，并设置 `info_completed=1`
- 若该学生无 `student_info` 记录：全量插入 CSV 中的数据，设置 `info_completed=1`

**第八步：** 导入完成后，系统显示成功数量、失败数量及前 10 条错误信息。

> ⚠️ **注意：**
> - 导入前账户必须已存在（先创建账户，再导入详细信息）
> - 日期格式必须正确，否则该行会报错
> - 导入操作会被记录在操作日志中

## 第八章 系统设置

### 8.1 进入系统设置页面

**第一步：** 点击左侧菜单中的【系统设置】选项。

**第二步：** 页面显示系统设置界面，顶部有五个标签页：**学院 / 年级 / 班级 / 政治面貌 / 发展时间**。

**重要说明：** 系统设置中的选项数据是学生填写信息时下拉框的来源。例如，学院选项中添加了"生命科学与化学学院"，学生填写民族时的学院下拉框才会出现该选项。因此，**设置的选项必须准确，并与实际情况一致**。

### 8.2 管理学院选项

**添加学院：**

**第一步：** 点击顶部【学院】标签页，进入学院选项管理界面。

**第二步：** 在页面中找到新增输入框（通常在选项列表顶部或底部，带有"添加"或"新增"字样）。

**第三步：** 在输入框中输入新学院的名称（如"生命科学与化学学院"）。

**第四步：** 点击【添加】（或【保存】）按钮。

**第五步：** 系统保存新学院，选项立即出现在列表中，学生填写信息时的学院下拉框随即包含该选项。

---

**编辑学院名称：**

**第一步：** 在学院列表中，找到需要修改的学院选项所在行。

**第二步：** 点击该行右侧的【编辑】按钮（铅笔图标）。

**第三步：** 该行变为可编辑状态，修改学院名称。

**第四步：** 点击【保存】按钮（或按回车键），系统更新该学院名称。

> ⚠️ **注意：** 修改学院名称后，已使用旧名称填写信息的学生数据**不会自动更新**，仍保留旧名称。

---

**删除学院：**

**第一步：** 在学院列表中，找到需要删除的学院选项所在行。

**第二步：** 点击该行右侧的【删除】按钮（垃圾桶图标）。

**第三步：** 系统弹出确认对话框，点击【确认删除】。

**第四步：** 该学院选项从列表中删除，学生填写信息时的下拉框不再包含该选项。

> ⚠️ **注意：** 删除学院选项不会删除已使用该学院名称的学生数据，仅影响后续填写。

### 8.3 管理年级选项

操作步骤与"管理学院选项"完全相同，切换到【年级】标签页后：

- **添加年级：** 输入年级名称（如"2026级"），点击【添加】
- **编辑年级：** 点击【编辑】按钮，修改名称后点击【保存】
- **删除年级：** 点击【删除】按钮，确认删除

> ⚠️ **注意：** 年级名称的格式建议统一（如统一使用"XXXX级"格式），便于系统按年级筛选和排序。

### 8.4 管理班级选项

切换到【班级】标签页，操作步骤与学院、年级相同：

- **添加班级：** 输入班级名称（如"生物3班"），点击【添加】
- **编辑班级：** 点击【编辑】，修改名称后保存
- **删除班级：** 点击【删除】，确认后删除

### 8.5 管理政治面貌选项

切换到【政治面貌】标签页，操作步骤相同：

- **添加政治面貌：** 输入政治面貌名称（如"群众"），点击【添加】
- **编辑：** 修改名称并保存
- **删除：** 删除不需要的政治面貌选项

> ⚠️ **注意：** 政治面貌选项与学生端的升级规则密切相关。系统内置了"入党积极分子"→"中共预备党员"→"中共党员"的升级路径，请不要随意删除这三个核心选项。

### 8.6 管理发展时间

**发展时间**是一种特殊的系统选项，用于记录特定年度的党员发展关键时间节点（如"2024年入党积极分子确认日期"）。学生填写信息时，系统会根据发展时间配置自动填入部分只读字段（如"确定积极分子时间"）。

**发展时间的存储格式：** `名称|日期`（竖线分隔），例如：`2024年积极分子确认|2024-03-15`

---

**添加发展时间：**

**第一步：** 切换到【发展时间】标签页。

**第二步：** 在新增区域，找到两个输入框：一个输入**时间名称**，一个输入**日期**。

**第三步：** 在时间名称输入框中输入名称（如"2025年积极分子确认"）。

**第四步：** 在日期输入框中选择或输入日期（格式：YYYY-MM-DD，如"2025-03-20"）。

**第五步：** 点击【添加】按钮，系统以"名称|日期"格式将数据保存到数据库，新条目出现在列表中。

---

**编辑发展时间：**

**第一步：** 在发展时间列表中，找到需要修改的条目所在行。

**第二步：** 点击该行的【编辑】按钮，该行变为可编辑状态，显示名称和日期两个独立输入框。

**第三步：** 修改名称或日期（或同时修改两者）。

**第四步：** 点击【保存】按钮，系统更新数据。

---

**删除发展时间：**

**第一步：** 在列表中找到需要删除的发展时间条目。

**第二步：** 点击【删除】按钮，确认后删除。

> ⚠️ **注意：**
> - 发展时间条目一旦被学生填写信息时引用，删除不会影响已填入的学生数据
> - 建议每年新增一条对应年度的发展时间配置，用于新一年度的学生信息填写
> - 同一年度通常需要配置多条发展时间（如积极分子确认日期、预备党员确认日期等）

### 8.7 拖拽排序操作步骤

系统设置中所有选项均支持拖拽排序，排序决定了学生填写信息时下拉框中选项的显示顺序：

**第一步：** 在任一选项列表（学院/年级/班级/政治面貌/发展时间）中，将鼠标移动到需要移动的选项行左侧的拖拽图标（通常显示为"⋮⋮"或"≡"符号）上。

**第二步：** 鼠标变为移动光标（十字箭头或抓手形状）。

**第三步：** 按住鼠标左键，将该行拖拽到目标位置（上方或下方）。

**第四步：** 松开鼠标，选项移动到新位置。

**第五步：** 系统自动保存新的排序（拖拽完成后立即提交到服务器），**无需手动点击"保存排序"按钮**。

> ⚠️ **注意：** 排序调整后，学生下次打开下拉框时即可看到新的选项顺序。建议将最常用的选项排在前面，方便学生选择。

| 操作类型 | 徽章颜色 | 操作说明 |

| 退出（logout） | 灰色 | 用户退出登录 |
| 创建（create） | 绿色 | 新增账户、创建数据等 |
| 更新（update） | 黄色 | 修改信息、学籍变动、系统设置修改等 |

| 导入（import） | 深蓝色 | 批量导入账户或学生信息 |
| 审核（audit） | 蓝色 | 审核通过或拒绝学生信息修改申请 |
| 密码（password） | 黄色 | 重置密码、修改密码等操作 |

### 9.4 日志字段含义说明

每条日志记录包含以下字段：

| 字段名称 | 含义说明 |
|----------|----------|
| 操作时间 | 该操作发生的具体时间（精确到秒） |
| 操作用户 | 执行操作的用户名和姓名 |
| 用户角色 | 操作用户的系统角色（student/teacher/admin/superadmin） |
| 操作类型 | 操作的类别（见上表） |
| 操作描述 | 对本次操作的详细文字描述（包含操作对象、变更内容等） |
| IP 地址 | 执行操作时的客户端 IP 地址 |

> ⚠️ **注意：** 
> - 日志中的敏感数据（如身份证号、手机号）已经过脱敏处理，以"*"代替部分位（如身份证号显示为"110***...***1234X"）
> - 日志为只读内容，不可修改或删除
> - 系统保留最近 30 天的日志数据，超期日志会自动清理

## 第十二章 常见问题 FAQ

### 12.1 账户无法登录怎么处理

**问题：** 某位学生反映账户无法登录。

**排查步骤：**

1. **检查账户状态：** 在账户管理页面搜索该学生，查看账户状态是否为"禁用"——若是，点击【启用】恢复
2. **检查是否已毕业：** 查看该学生账户是否标记为已毕业（`is_graduated=1`）——已毕业学生无法登录，若误操作毕业需联系超级管理员或数据库管理员处理
3. **密码问题：** 若学生忘记密码，在账户管理页面点击【重置密码】，将密码重置为用户名（学号），告知学生重新登录后修改密码
4. **IP封禁：** 若学生因多次输错密码被系统封锁IP，短时间内无法登录，需等待封禁自动解除（通常几分钟至数小时）或联系超级管理员解封

### 12.2 批量导入报错怎么办

**问题：** 批量导入账户或学生信息时出现错误。

**常见错误及处理方法：**

| 错误提示 | 原因 | 解决方法 |
|----------|------|----------|
| "用户名已存在" | 该学号的账户已创建 | 导入账户时跳过已有账户，只导入新账户 |
| "学院不存在" | 填写的学院名称与系统设置中不一致 | 检查系统设置中的学院名称，确保 CSV 中的名称完全一致（区分全角/半角、有无空格） |
| "年级不存在" / "班级不存在" | 同上 | 同上，检查年级/班级名称 |
| "身份证号格式错误" | 18位格式不正确 | 检查身份证号是否为18位，最后一位X是否正确 |
| "日期格式错误" | 日期格式不符合YYYY-MM-DD | 检查 CSV 中的日期列，确保格式为"2024-03-15"而非"2024/03/15" |
| "文件编码错误" | CSV 文件编码问题 | 在 Excel 中另存为"CSV UTF-8(带BOM)"格式后重新上传 |

**通用处理步骤：**

1. 查看系统返回的错误信息，找出第几行出现了什么错误
2. 打开 CSV 文件，定位到对应行，修正错误内容
3. 重新上传修正后的文件（可只包含失败的行，成功的行不会重复处理）

### 12.3 审核列表未自动刷新怎么办

**问题：** 信息审核页面的待审核列表没有自动更新，需要手动刷新页面。

**解决方法：**

1. **检查网络连接：** 确保电脑联网正常，自动刷新需要持续访问服务器
2. **浏览器兼容性：** 确保使用推荐的浏览器（Chrome、Edge、Firefox），且版本不过于老旧
3. **强制刷新页面：** 按 `Ctrl + F5`（Windows）或 `Cmd + Shift + R`（Mac）强制刷新
4. **重新登录：** 退出登录后重新进入信息审核页面
5. **联系技术支持：** 若以上均无效，通过反馈系统提交问题

### 12.4 系统设置修改后学生端不生效怎么办

**问题：** 在系统设置中添加了新的班级或年级，但学生填写信息时下拉框中没有出现新选项。

**解决方法：**

1. **等待缓存刷新：** 系统对部分数据有短时间缓存（约5分钟），新增选项可能需要几分钟后才在学生端生效
2. **让学生刷新页面：** 告知学生在填写信息页面按 `F5` 或 `Ctrl + R` 刷新页面，重新加载选项数据
3. **检查设置是否保存成功：** 在系统设置页面，确认新增的选项确实出现在列表中（若没有，说明添加操作未成功，需重新添加）
4. **检查选项是否启用：** 确认新增选项的状态为"启用"（部分情况下新增选项默认为禁用状态）
